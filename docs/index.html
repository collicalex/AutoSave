Description

AutoSave is a simple portable application to automatically save (backup) your computer files to your external storage drive.

It does only incremental backup: all new files will be saved; but files which do not more exist on your local computer will not be deleted from the backup (usb drive).

Tips

Autosave can manage both absolute and relative path in source and destination directories (see next section for an example used).

How I use it

1 / 2
I've put the AutoSave.jar in my dedicated external usb disc (in a dedicated folder).
Inside it, I've created a Backup directory where I will save my files.
2 / 2
Then I configure it by using absolute path for my source (internal drive);
and relative path in destination to save in my current usb drive.

I've configured it with

  • absolute path for source directories,
  • but with relative path for destination.

Because, my local (internal) drive will never change its letter (access path). But my operating system will attribute a "random" letter for my (external) usb drive. The first one usb drive connected will be "E:", then the second one will be "F:"... So if I've already connected a usb key, then my backup usb drive will have the letter F:, but if I've connected it alone, it will have the letter E:. And so, with absolute path, it will not work!

Warning

Under Windows 7 (and maybe newer version), there are some "hidden" folder in "Documents" (which is in fact "My Document"). These hidden folder are some kind of link, not opennable to : My Music, My Pictures and My Video. If you want to backup your "My document" directory, you must need to add to ignore list:

  • C:\Users\your_name\Documents\My Pictures
  • C:\Users\your_name\Documents\My Videos
  • C:\Users\your_name\Documents\My Musics

In french, it's:

  • C:\Users\your_name\Documents\Mes images

File Encryption

Autosave can use encryption to crypt (and decrypt) both file content and filename.

It does it via AES encryption method (which is fast and secure) in GCM mode. These both mixed together result in an authenticated encryption algorithm designed to provide both data authenticity (integrity) and confidentiality.

But as it's software encryption, it can be slow. Another solution to crypted your files is to used a hardware encrypted drive which are now not really expensive (see the Western Digital My Passport (for Mac)).

External libraries

Autosave used some external libraries:

  • Bouncing Castle
    Crypotography library faster than native java (jre) one.
  • Offset248
    To save encrypted filename.
  • MigLayout
    A layout manager library.
  • Java Native Access (JNA)
    To be able to stop the operating system standby during copy.
