Description

AutoSave is a simple portable application to automatically save (backup) your computer files to your external storage drive.

It does only incremental backup: all new files will be saved; but files which do not more exist on your local computer will not be deleted from the backup (usb drive).

How to use it

1 / 15
At first run, nothing is configured.
2 / 15
To configure a new folder or file to backup, use the + button.
3 / 15
A new empty entry is created. The source textfield is yellow as it's empty (or doest not point to an existing folder/file).
4 / 15
You have to define source folder (or file), destination folder (or file),
recursive or not (i.e. save sub folders), and if you want ignore some folders or files.
5 / 15
You can add as many entries as you want.
6 / 15
Don't forget to save your configuration. The save button will be red if any change.
Tips: When you close the application, if any change, a popup window will invite you to save your new configuration.
7 / 15
By default, the program will try to load autosave.ini configuration file in the same folder of the AutoSave.jar file.
Then it's hightly recommanded to save your configuration file as this.
8 / 15
Once saved, the button will be back black, and the configuration file path is displayed in the textfield.
9 / 15
Before running a real backup, you can activate the simulation mode by checking the option in top left.
10 / 15
You can run a backup (simulated or not), by clicking either on an individual run backup button (it will run the backup only for the related entry), either on the backup all button, to backup all entries.
11/ 15
Once the backup (simulated or real) is done, it will popup the number of new file saved.
New files are the one with a modification date on your sources greater to the one in your destination.
12/ 15
In top right part, some statistics are displayed. In the log part (main part at right), firstly is displayed the number of files per source directory. Then during the backup, if a file is skip, it's displayed in grey, and you can view that the copy is tag as SIMU.
13/ 15
Once you have verified in the simulated log that the source and destination are correct,
you can uncheck the simulation option, and run the real backup with the backup all button.
14/ 15
During the backup, you can view in statistics panel the estimated end time
and in the log part that the copy is now tag as COPY (the copy is really done).
15 / 15
Obviously in this test to create these screenshot, when I change one file in my source (edit text file and save),
and run again the backup, the software found 1 new file to save!

Tips

Autosave can manage both absolute and relative path in source and destination directories (see next section for an example used).

How I use it

1 / 2
I've put the AutoSave.jar in my dedicated external usb disc (in a dedicated folder).
Inside it, I've created a Backup directory where I will save my files.
2 / 2
Then I configure it by using absolute path for my source (internal drive);
and relative path in destination to save in my current usb drive.

I've configured it with

  • absolute path for source directories,
  • but with relative path for destination.

Because, my local (internal) drive will never change its letter (access path). But my operating system will attribute a "random" letter for my (external) usb drive. The first one usb drive connected will be "E:", then the second one will be "F:"... So if I've already connected a usb key, then my backup usb drive will have the letter F:, but if I've connected it alone, it will have the letter E:. And so, with absolute path, it will not work!

Warning

Under Windows 7 (and maybe newer version), there are some "hidden" folder in "Documents" (which is in fact "My Document"). These hidden folder are some kind of link, not opennable to : My Music, My Pictures and My Video. If you want to backup your "My document" directory, you must need to add to ignore list:

  • C:\Users\your_name\Documents\My Pictures
  • C:\Users\your_name\Documents\My Videos
  • C:\Users\your_name\Documents\My Musics

File Encryption?

For the moment (release 2.0) Autosave does not support file encryption.

You can use an hardware encrypted drive which are now not really expensive (see the Western Digital My Passport (for Mac))

In the next release, Autosave will do it via AES encryption (which is fast and secure) in GCM mode. These both mixed together result in an authenticated encryption algorithm designed to provide both data authenticity (integrity) and confidentiality.
